Quality Improvement Starter Kit

A Lean Six Sigma tool-selection guide, root cause analysis checklist, and quality tools cheat sheet.

1. Lean Six Sigma Tool-Selection Guide

Use this table to pick the right tool for the problem you're facing.

SituationUse this tool
You don't know why a problem keeps happening5 Whys or Fishbone Diagram
You have many problems and need to prioritizePareto Chart (80/20 rule)
A process seems inconsistent day to dayControl Chart
You need to map every step of a processProcess Map / Value Stream Map
You want to prevent failures before they happenFMEA (Failure Mode and Effects Analysis)
You need to compare two or more groups statisticallyHypothesis Test or ANOVA
You're running a full improvement projectDMAIC (Define, Measure, Analyze, Improve, Control)

2. Root Cause Analysis Checklist

  • Define the problem in one clear sentence — avoid vague statements.
  • Gather data before assuming a cause. Don't guess.
  • Ask "why" at least five times, following each answer to the next question.
  • Consider multiple categories: people, process, equipment, materials, environment.
  • Confirm the root cause with data, not just a hunch.
  • Assign clear ownership for the fix — a root cause without an owner recurs.
  • Follow up after the fix to confirm the problem doesn't come back.

3. Quality Tools Cheat Sheet

ToolWhat it does
5 WhysTraces a symptom back to its root cause by repeatedly asking "why."
Fishbone DiagramOrganizes potential causes into categories to spot patterns.
Pareto ChartRanks problems by frequency or impact so you fix the biggest ones first.
Control ChartDistinguishes normal process variation from a real, actionable shift.
FMEAScores potential failure modes by severity, likelihood, and detectability.
