Adding a New Lesson to UpSkill Sprint

A simple guide for future you.

You don't need to know how to code to add a lesson. Every lesson follows the same simple format, and there are two ways to write one. Once it's written, just tell your design assistant (Claude) the details below and it will build the page and add it to the site for you.

Step 1 — Write the lesson

Use this format every time, so lessons stay consistent:

  • Title — short and clear, e.g. "How to Use the 5 Whys"
  • What you will learn — one sentence
  • Why this matters at work — 2–3 sentences on the real value
  • Simple explanation — teach the idea in plain language
  • Real workplace example — how it shows up on the job
  • Try it yourself — a small practice activity
  • Download (optional) — a template, checklist, or dataset

You can write this in any format that's easiest for you:

  • A Word document or Google Doc, following the sections above
  • Plain notes or bullet points — Claude can turn these into a finished lesson page
  • A ready-made HTML file, if you start from the Lesson Template file already in the project

Step 2 — Tag it

Every lesson gets three tags. These decide where it shows up on the site and let visitors filter later.

Tag Options
Skill Track Lean Six Sigma, Data Analysis, Machine Learning, Quality Engineering, Project Management
Level Beginner, Intermediate, Advanced
Estimated Time e.g. 12 min, 15 min, 25 min

Step 3 — Ask Claude to add it

Send your lesson content (doc, notes, or file) and say something like:

"Here's a new lesson called '[Title]'. It's [Beginner/Intermediate/Advanced], about [X] minutes, and belongs in the [Skill Track] track. Please create the lesson page from the Lesson Template and add a card for it to the Lessons section on the homepage."

Claude will duplicate the Lesson Template, fill it in with your content, save it as its own file, and add a matching card to the Lessons section so visitors can find it.

Adding a tool, dataset, or case study instead

Same idea — send the file or description and ask Claude to add it to the Tools & Templates or Case Studies section, with a short title and description. No special format required for these.

The short version

Write once, tag once, send it over. You handle the content — Claude handles the page and the placement.
