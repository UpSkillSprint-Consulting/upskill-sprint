UpSkill Sprint Consulting — Internal Guide

Adding a New Lesson to UpSkill Sprint

The site is plain HTML/CSS hosted on Netlify via GitHub — no build step, no framework.

You don't need to know how to code to add a lesson. Write it once, tag it, and tell your design assistant (Claude) the details below — it will build the page and add it to the site for you.

Step 1 — Write the lesson

Every lesson follows the same design and structure, built from lesson-template.html and lessons-theme.css:

  • Eyebrow + title — level and read time, e.g. "Beginner · 10 min read"
  • Topic badge — a colored pill under the intro showing the Skill Track (e.g. badge-lss, badge-quality)
  • By the end, you should be able to — 2–3 checklist objectives
  • Why this matters at work — 2–3 sentences on the real value
  • Simple explanation — plain-language teaching, with a highlighted formula/rule callout
  • Real workplace example — a numbered step-by-step walkthrough in an example box
  • Common mistake — a warning callout on what learners often get wrong
  • Try it yourself — a small practice activity in a green callout
  • Check your understanding — a one-question click-to-check quiz

You can send this content in any format that's easiest for you — a doc, plain notes, or bullet points — and Claude will build the page in this structure. lesson-template.html in the project root is a ready-to-copy blank version.

Design reference

The full color system, badge colors per track, and copy-ready component markup (callouts, step-list, quiz, objectives) live in upskillsprint_lesson_theme_reference.html. Every new lesson should reuse these components rather than introducing new ones.

Step 2 — Tag it

Every lesson gets three tags. These decide where it shows up on the site.

Tag Options
Skill Track Lean Six Sigma, Quality Tools, Statistics, Excel, Power BI, SQL, Process Improvement, Project Management, Business Analysis
Level Beginner, Intermediate, Advanced
Estimated Time e.g. 8 min, 12 min, 15 min

Step 3 — Ask Claude to add it

Send your lesson content (doc, notes, or file) and say something like:

"Here's a new lesson called '[Title]'. It's [Beginner/Intermediate/Advanced], about [X] minutes, and belongs in the [Skill Track] track. Please create the lesson page from the template and add a card for it to the Lessons page."

Claude will duplicate lesson-template.html into the lessons/ folder, fill it in with your content, and add a matching card to the right track section on lessons.html (each track section has an id matching a chip at the top, e.g. id="quality-tools").

If it's a strong beginner lesson, Claude can also feature it in the "Featured Lessons" section of index.html.

Step 4 — Publish

  1. Commit the new lesson file — plus lessons-theme.css if it's new to your repo — along with the edited lessons.html (and index.html if featured), to the GitHub repo.
  2. Netlify redeploys automatically within a minute or two.
  3. Add the new page's URL to sitemap.xml so search engines find it faster.

Adding a tool, dataset, or case study instead

Same idea — send the file or description and ask Claude to add it to the relevant section, with a short title and description. No special format required.

The short version

Write once, tag once, send it over. You handle the content — Claude handles the page and the placement.
