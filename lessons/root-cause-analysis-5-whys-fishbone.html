Root Cause Analysis: 5 Whys vs Fishbone Diagram

What you will learn: two simple ways to trace a problem back to its real cause, and when to use each.

Why this matters at work

Fixing symptoms instead of causes means the same problem keeps coming back. Root cause analysis stops the cycle by forcing you to ask "why" until you hit something you can actually act on.

Simple explanation

The 5 Whys is just asking "why did that happen?" five times in a row, each answer feeding the next question. A Fishbone (Ishikawa) diagram does the same thing but branches out into categories — people, process, equipment, materials — so you can spot causes across multiple angles at once.

Real workplace example

A machine keeps jamming. Five Whys: it jams because of debris → debris builds up because the filter isn’t cleaned → the filter isn’t cleaned because no one owns that task → no one owns it because it was never assigned after a staffing change. That’s the real fix: assign ownership, not just clean the machine once.

Try it yourself

Pick a recurring annoyance at work. Ask "why" five times, writing down each answer. See how far the real cause is from your first guess.
