Strategic sequence

1. AffinityGroup raw ideas into natural themes.
2. InterrelationshipIdentify influential drivers and outcomes.
3. TreeBreak a broad goal into executable work.
4. PrioritizationRank choices against agreed criteria.
5. MatrixClarify relationships, roles, and gaps.
6. PDPCAnticipate failure and prepare countermeasures.
7. Activity NetworkSequence tasks and expose the critical path.
Boundary rule: use the basic QC tools while the problem still needs evidence. Use the management and planning tools when the cause is credible and the team must organize and execute a response.

PDPC challenge

Plan: add a temperature alarm.

What could go wrong? nuisance alarms create alarm fatigue.

Countermeasure: validated alarm limits, escalation logic, calibration checks, and response training.

PDPC is plan-risk analysis. It is not a replacement for process FMEA where a formal risk method is required.

Integrated planning challenge

A steel mill has confirmed that unstable reheat-furnace practice is contributing to low-yield-strength failures. Your job is to turn the finding into a defensible implementation plan.

  1. Use Affinity to organize ideas from Operations, Maintenance, Quality, and Metallurgy.
  2. Use the Interrelationship Digraph to identify the strongest drivers.
  3. Use a Tree Diagram to convert the selected strategy into work packages.
  4. Use a Prioritization Matrix to select the first actions.
  5. Use a Matrix Diagram to assign relationships and ownership.
  6. Use PDPC to anticipate implementation failure.
  7. Use the Activity Network to sequence work and establish the critical path.

Knowledge check

The activity network reveals that the deadline is impossible. What is the disciplined response?

Common failure modes

  • Predefining affinity categories before the team generates ideas.
  • Treating arrow counts as proof of causality.
  • Stopping the tree one level too early.
  • Changing prioritization weights after seeing the scores.
  • Assuming ownership is obvious instead of defining it.
  • Skipping PDPC as “negative thinking.”
  • Scheduling before the task list and dependencies are complete.
